# vLLM Metrics High Priority Fixes - Summary

## Date: 2026-02-05

## Issues Fixed

### 1. ✅ FIXED: Inconsistent Counter Metric Treatment for Token Totals

**Issue:** Token counter metrics were missing or incorrectly handled
- `Prompt Tokens Total` and `Generation Tokens Total` metrics expected by frontend were not being generated by backend
- If they existed, they would have shown absolute counter values since process start instead of counts during the selected time window

**Fix:** Added proper counter metrics with `increase()` function in `src/core/metrics.py` (lines ~893-902):

```python
# Phase 2: Total token counts (COUNTER - use increase() to show total during time window)
# These are separate from "Prompt/Output Tokens Created" shown in Key Metrics
if "vllm:prompt_tokens_total" in vllm_metrics:
    metric_mapping["Prompt Tokens Total"] = "sum(increase(vllm:prompt_tokens_total[5m]))"
elif "vllm:request_prompt_tokens_sum" in vllm_metrics:
    metric_mapping["Prompt Tokens Total"] = "sum(increase(vllm:request_prompt_tokens_sum[5m]))"

if "vllm:generation_tokens_total" in vllm_metrics:
    metric_mapping["Generation Tokens Total"] = "sum(increase(vllm:generation_tokens_total[5m]))"
elif "vllm:request_generation_tokens_sum" in vllm_metrics:
    metric_mapping["Generation Tokens Total"] = "sum(increase(vllm:request_generation_tokens_sum[5m]))"
```

**Impact:** Values now correctly show token counts during the selected time window (1h, 6h, etc.) instead of total since vLLM started

### 5. ✅ FIXED: Cache Gauge Metrics Missing from GAUGE_METRICS List

**Issue:** Cache percentage metrics were not in the `GAUGE_METRICS` list, so they wouldn't get `max_over_time()` wrapping for sparklines
- This meant brief cache usage spikes could be missed in visualizations

**Fix:** Added cache metrics to `GAUGE_METRICS` list in `src/core/metrics.py` (line ~163):

```python
GAUGE_METRICS = [
    "GPU Usage (%)",
    "GPU Utilization (%)",
    # ... other GPU metrics
    # Cache metrics (also gauges that can spike briefly)
    "Kv Cache Usage Perc",
    "Gpu Cache Usage Perc",
]
```

**Impact:** Cache usage sparklines now correctly show peak values during each sampling interval, preventing missed spikes
